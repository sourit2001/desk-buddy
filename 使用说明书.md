# 桌宠程序使用说明书

## 1. 程序简介

本程序是一个基于 Tauri 2 + React 的桌面宠物应用。它可以在桌面上显示透明置顶的小窗口，支持图片桌宠和 MMD 模型桌宠，并提供拖动、互动、自动移动、状态显示和 OpenAI-compatible 对话能力。

主要功能：

- 上传 PNG/JPG/WebP 图片作为桌宠形象
- 上传 MMD 模型包或 `.pmx` / `.pmd` 模型，并可加载 `.vmd` 动作
- 支持多个桌宠配置，能分别设置名称、性格和口头禅
- 支持置顶显示、窗口尺寸调整、空闲自由移动
- 支持待机、点击、说话、表情、多图帧动画
- 支持右键互动菜单：摸摸、喂食、休息、玩一下、走走路、打招呼、点点头、聊天
- 支持配置 OpenAI-compatible Chat Completions API，让桌宠回复聊天内容

## 2. 运行环境

开发运行需要安装：

- Node.js
- npm
- Rust 工具链
- Tauri 2 所需系统依赖

如果只是使用已经打包好的应用，不需要安装以上开发环境，直接打开应用程序即可。

## 3. 开发启动

在项目根目录执行：

```bash
npm install
npm run tauri:dev
```

启动后会出现桌宠窗口。第一次没有配置形象时，窗口中会显示“打开设置”的占位内容。

也可以只启动浏览器调试页面：

```bash
npm run dev
```

浏览器调试模式不具备完整桌面窗口能力，例如真实置顶、窗口拖动和本机配置保存会与桌面应用模式不同。

## 4. 打包构建

在项目根目录执行：

```bash
npm run tauri:build
```

构建产物会输出到 Tauri 的打包目录中。macOS 通常生成 `.app` / `.dmg`，Windows 通常生成对应安装包或可执行文件。跨平台包需要在对应操作系统上构建。

## 5. 首次使用流程

1. 启动程序。
2. 点击桌宠窗口中的“设置”按钮，或双击桌宠打开设置窗口。
3. 在“形象”区域配置桌宠。
4. 上传图片，或切换到 MMD 模式并选择模型。
5. 根据需要调整窗口大小、置顶、动画和自由移动。
6. 点击“保存”或“保存设置”。
7. 点击“显示桌宠”回到桌宠窗口。

## 6. 图片桌宠设置

在设置窗口的“形象”区域中，将“显示模式”设为“图片桌宠”。

可用操作：

- “当前宠物名称”：修改桌宠显示名称。
- “性格”：选择温和、活泼、酷酷的、黏人。性格会影响闲聊文案和互动回复。
- “口头禅”：可选。填写后，部分互动会优先显示该口头禅。
- “上传时自动抠图”：上传普通图片时尝试自动去掉背景。
- “抠图容差”：控制抠图强度。背景接近纯色时效果较好；复杂照片背景可能不理想。
- “添加图片”：上传一张或多张图片作为桌宠形象。
- “清空”：清除当前桌宠的全部图片。
- 缩略图右上角删除按钮：删除单张图片。

建议使用透明 PNG。若使用普通 JPG/WebP，建议图片背景尽量纯色，并开启“上传时自动抠图”。

## 7. 多图帧动画

如果一个桌宠上传了多张图片，可以在“窗口与动画”区域开启“播放多图帧动画”。

相关设置：

- “启用动画”：总开关，关闭后桌宠动画和表情效果会减少。
- “动画强度”：控制动作幅度。
- “播放多图帧动画”：按顺序或随机切换多张图片。
- “启用表情变化”：在图片桌宠上叠加简单表情效果。
- “偶尔随机跳帧”：开启后不会完全按顺序播放，会偶尔跳到随机图片。
- “帧间隔（秒）”：控制图片切换速度。

如果只上传一张图片，多图帧动画不会产生明显效果。

## 8. MMD 桌宠设置

在设置窗口的“形象”区域中，将“显示模式”设为“MMD 模型”。

可用操作：

- “选择模型包”：选择 `.zip`、`.pmx` 或 `.pmd` 模型文件。
- “选择动作 .vmd”：选择已解压的 `.vmd` 动作文件。
- “MMD 缩放”：调整模型显示大小。
- “清空 MMD”：移除当前桌宠的 MMD 模型和动作。

注意事项：

- 动作文件必须选择 `.vmd`，不要直接选择 zip 动作包。
- `camera.vmd` 或 `カメラ.vmd` 通常是镜头动作，不适合作为桌宠动作。
- 如果没有选择动作，程序会使用内置的轻动作效果。
- 如果没有选择模型，MMD 模式会先显示 3D 预览。

## 9. 窗口与移动设置

在“窗口与动画”区域中可以调整桌宠窗口行为。

设置项说明：

- “宽度 / 高度”：调整桌宠窗口大小。
- “置顶显示”：开启后桌宠窗口会保持在其他窗口上方。
- “空闲时自由移动”：开启后，桌宠在空闲时会自动移动。
- “移动范围”：设置桌宠移动区域，包括屏幕四边、上下边缘、左右边缘、单侧边缘、屏幕中间和全屏随机。
- “移动间隔（秒）”：每隔多久尝试移动一次。
- “移动耗时（秒）”：每次移动持续多久。

自由移动不会在聊天输入框打开时触发，也不会在桌宠正在执行其他动作时触发。

## 10. 桌宠窗口操作

桌宠窗口提供以下操作：

- 左键点击桌宠：摸摸桌宠，增加亲密度，消耗少量精力。
- 按住桌宠拖动：移动桌宠窗口位置。
- 双击桌宠：打开设置窗口。
- 右键桌宠：打开互动菜单。
- 顶部“互动”按钮：打开互动菜单。
- 顶部“聊天”按钮：打开聊天输入框。
- 顶部“设置”按钮：打开设置窗口。
- 顶部“退出”按钮：退出程序。

桌宠窗口会显示当前桌宠名称、亲密度和精力。不同互动会改变亲密度和精力。

## 11. 互动菜单

右键桌宠或点击“互动”按钮后，可以选择：

- “摸摸”：增加亲密度。
- “喂食”：增加精力，略微增加亲密度。
- “休息”：恢复较多精力。
- “玩一下”：增加亲密度，但消耗精力。
- “走走路”：触发走路动作。
- “打招呼”：触发打招呼动作。
- “点点头”：触发点头动作。
- “聊天”：打开聊天输入框。

如果桌宠设置了口头禅，部分互动会优先显示口头禅。

## 12. 对话 API 配置

在设置窗口的“对话 API”区域填写：

- “Base URL”：API 地址，例如 `https://api.openai.com/v1`。
- “Model”：模型名称，例如 `gpt-4.1-mini`。
- “API Key”：接口密钥。
- “系统提示词”：控制桌宠聊天风格和身份。

程序会调用 OpenAI-compatible `/chat/completions` 接口。配置完成并保存后，点击桌宠窗口的“聊天”按钮即可输入消息。

聊天时，程序会把当前桌宠名称、性格、亲密度、精力和口头禅加入提示词中，让回复更符合当前桌宠状态。

## 13. 配置保存

在桌面应用模式下，配置保存在本机应用数据目录中。浏览器调试模式下，配置会回退保存到浏览器 `localStorage`。

API Key 只保存在本机。请不要把包含个人密钥的配置文件提交到公开仓库。

## 14. 常见问题

### 上传图片后背景没有去干净

自动抠图适合纯色或近似纯色背景。可以尝试：

- 使用透明 PNG
- 开启“上传时自动抠图”
- 调高或调低“抠图容差”
- 换用背景更干净的图片

### 多图动画没有播放

请确认：

- 当前桌宠至少上传了两张图片
- “启用动画”已开启
- “播放多图帧动画”已开启
- “帧间隔（秒）”没有设置得过大

### 聊天按钮不能发送

请确认：

- 已填写 API Key
- Base URL 和 Model 正确
- 当前网络可以访问对应接口
- API 服务兼容 OpenAI Chat Completions 格式

### MMD 动作无法加载

请确认：

- 选择的是 `.vmd` 文件
- 动作包已经先解压
- 没有选择 `camera.vmd` 或 `カメラ.vmd`
- 模型文件和贴图文件完整

### 桌宠不会自动移动

请确认：

- “空闲时自由移动”已开启
- 当前没有打开聊天输入框
- 桌宠没有正在执行点击、说话或其他动作
- “移动间隔（秒）”没有设置得过长

## 15. 安全与限制

- 自动抠图是本地边缘背景识别，复杂背景效果有限。
- 通用动画不依赖骨骼或口型识别，不是 Live2D。
- MMD 模型和动作兼容性取决于模型资源本身。
- 对话能力依赖外部 OpenAI-compatible API，程序本身不内置大模型。
- API Key 仅本机保存，但仍应妥善保管。
